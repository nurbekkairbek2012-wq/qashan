## Проблема

**Один пользователь:** студент 17–20 лет с первым доходом — стипендия или подработка.

**Одна боль:** он берёт рассрочку «0-0-4». Потом вторую в другом магазине. Потом третью.
Каждая по отдельности выглядит безобидно — «всего 15 000 в месяц». Но совокупной картины
не видит никто: каждая рассрочка живёт в своём приложении. В какой-то месяц платежи молча
съедают весь доход.

Банк показывает, **сколько ты должен**. Никто не показывает, **когда ты не потянешь**.

## Решение

Qaryz отвечает на два вопроса:

1. **В каком месяце ты уйдёшь в минус** — помесячная симуляция денежного потока.
2. **Что будет, если взять ещё одну** — симулятор до покупки, а не разбор после.

Плюс считает **реальную эффективную ставку через IRR**: если в рассрочке зашита наценка,
она превращается в понятное «это X% годовых». Если наценки нет — модель честно показывает 0.

## Стек

| Слой | Технология | Почему |
|---|---|---|
| Frontend | React + Vite + Tailwind v4 | скорость разработки; тема CSS-first |
| БД / Auth / Storage | Supabase (PostgreSQL + RLS) | закрывает три слоя сразу |
| Серверная логика | Supabase Edge Functions (Deno) | ключ AI-API не попадает на клиент |
| AI | Claude API (vision) | распознавание скриншотов рассрочек |

Подробно — [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md): схема данных, модель расчёта,
обоснование решений.

## Роль ИИ

**LLM извлекает — код считает.** Модель распознаёт скриншот рассрочки и отдаёт черновой JSON.
Вся арифметика (график платежей, IRR, симуляция) — детерминированный код, покрытый тестами.
Языковая модель ошибается в арифметике, и доверять ей деньги нельзя.

Цепочка `parse-installment`: vision-извлечение → нормализация → валидация инварианта
(`платёж × срок ≈ сумма`) → расчёт. Расхождение не проглатывается молча — показывается
пользователю.

## Данные

**Синтетических данных в проекте нет.** Используем только реальные:

| Источник | Что берём |
|---|---|
| CustDev-интервью | настоящие рассрочки — с согласия, обезличенно |
| Публичные страницы товаров | пары «цена за наличные / цена в рассрочку», собранные **вручную** |
| Нацбанк РК | базовая ставка — база для сравнения |
| stat.gov.kz | медианный доход молодёжи по регионам |

Массового автоскрейпинга не ведём. Персональные финансовые данные — только с согласия
и обезличенно: в демо нет реальных имён и номеров.

## Открытый вопрос

Гипотеза «в рассрочке 0-0-4 зашита скрытая наценка» **не подтверждена и проверяется данными**.
В Kaspi Red комиссию платит продавец — переплаты для покупателя может не быть вовсе.
Результат проверки будет опубликован здесь честно, каким бы он ни оказался.

Боль от этого не зависит: невидимая совокупная нагрузка существует при любой ставке,
хоть при нулевой.

## Локальный запуск

Требуется Node.js 20+ (разрабатывалось на 22).

```bash
git clone https://github.com/nurbekkairbek2012-wq/<repo>.git
cd <repo>/frontend
npm install
npm run dev          # http://localhost:5173
```

Команды:

```bash
npm run dev       # dev-сервер
npm run build     # production-сборка в dist/
npm run preview   # предпросмотр сборки
npm run lint      # ESLint
```

Переменные окружения — см. `frontend/.env.example` (появится вместе с подключением Supabase).
Ключи в репозиторий не коммитятся.
